# AGENTS.md

## Repository purpose
Public website and Sparkle update feed for Notomo, a macOS app. Source repo: `https://github.com/natsustan/notomo-app`. Hosted via GitHub Pages at `https://natsustan.github.io/notomo-app/` — pushes to the default branch are published directly (no build step), so files in this repo map 1:1 to URLs under that origin (e.g. `appcast.xml` → `https://natsustan.github.io/notomo-app/appcast.xml`, `Notomo.zip` → `https://natsustan.github.io/notomo-app/Notomo.zip`, which is the URL Sparkle clients fetch).

## Structure
- `index.html` — Landing page (currently a placeholder).
- `appcast.xml` — Sparkle RSS feed advertising app updates. Each `<item>` describes a release with `sparkle:version` (build number), `sparkle:shortVersionString`, `sparkle:minimumSystemVersion`, and an `<enclosure>` pointing to the downloadable `.zip` with an `sparkle:edSignature` (EdDSA signature).
- `Notomo.zip` — Current packaged app build referenced by `appcast.xml`.
- `README.md` — One-line project description.

## Build / lint / test
There is no build system, package manager, or test suite. No commands to run. Validate XML manually with `xmllint --noout appcast.xml` if available.

## Releasing a new version
1. Replace `Notomo.zip` with the new build.
2. Add a new `<item>` at the top of `<channel>` in `appcast.xml` with updated `<title>`, `<pubDate>` (RFC 822), `sparkle:version`, `sparkle:shortVersionString`, `enclosure url`, `length` (file size in bytes; check with `wc -c Notomo.zip`), and a fresh `sparkle:edSignature` generated by Sparkle's `sign_update` tool against the new zip.
3. Keep `sparkle:minimumSystemVersion` accurate to what the app actually requires.

## Style
- Use 4-space indentation in `appcast.xml` (matches existing file); 2-space in HTML.
- Keep `index.html` minimal, semantic, and self-contained (no external assets unless necessary).
- Do not commit unrelated binaries; the only binary tracked is `Notomo.zip`.
